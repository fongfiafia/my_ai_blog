# 该不该用Windsurf

## 结论先行

- **和cursor一起使用**：cursor解决不了的问题，在windsurf上试试，也许有奇效
- **没必要专门学** ：操作方式和cursor差不多 （可以等我写几篇教程）
- **先用Cursor**：Cursor已经迭代很久，使用更加稳定，可以先根据我的教程先马上实践起来

## 吃瓜的我

关注AI编程的同学一定知道最近出了一个windsurf，网上视频非常多，很多人都说windsurf超越Cursor、比Cursor更加强大、抛弃Cursor吧！

我却一直保持围观状态。原因是我真的觉得Cursor普通模式就够用。

普通模式够用是因为我能够主动构建上下文，帮助cursor理解内容，所以我的代码bug偏少，就算出现bug我会提供更详细的错误信息给到cursor，减少鬼打墙问题，我觉得这点目前ai是很难自助做到的。	

我上周末分别使用agent和windsurf完成同一件事情，发现双方对于上下文的自动补充其实半斤八两，也印证了我这一点想法。

## 测试一下

我让cursor agent和windsurf都来做一件相同的事情：写一个windsurf的教程博客网站，风格模仿我现在这个博客。

为什么我挑这件事情呢？因为「模仿」、「参考」这四个字最考验二者自动获取上下文能力。它们需要先理解我这个项目，读懂我的框架、页面布局、i18n、目录结构等等，然后再基于这些来进行开发，查问题。

## 结果

最后我测试的结果：二者半斤八两，没有所谓的谁领先谁。都存在很多bug，反复无法解决。当然二者的区别就是犯的错误不同，bug不同，哈哈哈哈....

很有意思的是，当我把cursor写的问题，一个无论提问多少次cursor都无法解决的问题。交给windsurf来帮忙解决，windsurf却能够解决。

所以这两个东东你大可以搭配使用。左手cursor，右手windsurf，二者都打开相同项目，让它们互相竞争，谁解决了bug，谁就牛逼，没必要非谁不可。
